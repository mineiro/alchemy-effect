Async Workers let you write plain `async fetch` handlers — no Effect
runtime in the bundle. Alchemy bundles and deploys your file as-is while
still managing your infrastructure resources declaratively.

This guide walks through every binding type you can attach to an Async
Worker via the `bindings` prop.

:::note
For the Effect-based Worker pattern, see the [Effect Worker guide](/guides/worker).
For the full API reference, see the [Worker provider](/providers/cloudflare/worker).
:::

## How it works

<Steps>

1. **Declare resources** and pass them into the Worker's `bindings` prop
   in your `alchemy.run.ts`.

2. **Export a `WorkerEnv` type** using `Cloudflare.InferEnv` — this gives
   you a fully typed `env` object.

3. **Write a standard handler** (`export default { async fetch ... }`).

</Steps>

```typescript
// alchemy.run.ts
import * as Alchemy from "alchemy-effect";
import * as Cloudflare from "alchemy-effect/Cloudflare";
import * as Effect from "effect/Effect";

export type WorkerEnv = Cloudflare.InferEnv<typeof Worker>;

export const Worker = Cloudflare.Worker("Worker", {
  main: "./src/worker.ts",
  bindings: {
    // resources go here
  },
});

export default Alchemy.Stack(
  "MyApp",
  { providers: Cloudflare.providers() },
  Effect.gen(function* () {
    const worker = yield* Worker;
    return worker.url;
  }),
);
```

```typescript
// src/worker.ts
import type { WorkerEnv } from "../alchemy.run.ts";

export default {
  async fetch(request: Request, env: WorkerEnv) {
    // env is fully typed based on your bindings
    return new Response("Hello!");
  },
};
```

## Assets

When you set the `assets` prop on a Worker, Alchemy automatically adds
an `ASSETS` binding to the environment. You don't need to declare it in
`bindings` — it's implicit.

```typescript
// alchemy.run.ts
export const Worker = Cloudflare.Worker("Worker", {
  main: "./src/worker.ts",
  assets: {
    directory: "./public",
  },
});
```

Use `env.ASSETS.fetch(request)` to serve static files as a fallback:

```typescript
// src/worker.ts
import type { WorkerEnv } from "../alchemy.run.ts";

export default {
  async fetch(request: Request, env: WorkerEnv) {
    const url = new URL(request.url);

    if (url.pathname.startsWith("/api/")) {
      return new Response(JSON.stringify({ ok: true }), {
        headers: { "content-type": "application/json" },
      });
    }

    // fall through to static assets
    return env.ASSETS.fetch(request);
  },
};
```

You can also pass `assets` as a plain string path:

```typescript
export const Worker = Cloudflare.Worker("Worker", {
  main: "./src/worker.ts",
  assets: "./public",
});
```

## R2 Bucket

R2 provides zero-egress-fee object storage with an S3-compatible API.

```typescript
// alchemy.run.ts
export const Bucket = Cloudflare.R2Bucket("Bucket");

export type WorkerEnv = Cloudflare.InferEnv<typeof Worker>;

export const Worker = Cloudflare.Worker("Worker", {
  main: "./src/worker.ts",
  bindings: {
    Bucket,
  },
});
```

The `env.Bucket` binding exposes the standard R2 API — `get`, `put`,
`delete`, `list`, and `head`:

```typescript
// src/worker.ts
import type { WorkerEnv } from "../alchemy.run.ts";

export default {
  async fetch(request: Request, env: WorkerEnv) {
    const object = await env.Bucket.get("my-key");
    return new Response(object?.body ?? null, { status: object ? 200 : 404 });
  },
};
```

## D1 Database

D1 is Cloudflare's serverless SQLite database. Bind it to run SQL
queries at the edge.

```typescript
// alchemy.run.ts
export const DB = Cloudflare.D1Database("DB");

export type WorkerEnv = Cloudflare.InferEnv<typeof Worker>;

export const Worker = Cloudflare.Worker("Worker", {
  main: "./src/worker.ts",
  bindings: {
    DB,
  },
});
```

Use `env.DB` with the standard D1 API — `prepare`, `batch`, `exec`,
and `dump`:

```typescript
// src/worker.ts
import type { WorkerEnv } from "../alchemy.run.ts";

export default {
  async fetch(request: Request, env: WorkerEnv) {
    const { results } = await env.DB.prepare("SELECT * FROM users WHERE id = ?")
      .bind(1)
      .all();
    return Response.json(results);
  },
};
```

## Durable Objects

Durable Objects provide globally unique, stateful instances. Define the
DO class in your worker source file and reference it from the stack with
`Cloudflare.DurableObjectNamespace`. The `<Counter>` type parameter
ensures the stub is fully typed with the DO's public methods.

```typescript
// alchemy.run.ts
import type { Counter } from "./src/worker.ts";

export type WorkerEnv = Cloudflare.InferEnv<typeof Worker>;

export const Worker = Cloudflare.Worker("Worker", {
  main: "./src/worker.ts",
  bindings: {
    Counter: Cloudflare.DurableObjectNamespace<Counter>("Counter", {
      className: "Counter",
    }),
  },
});
```

```typescript
// src/worker.ts
import { DurableObject } from "cloudflare:workers";
import type { WorkerEnv } from "../alchemy.run.ts";

export default {
  async fetch(request: Request, env: WorkerEnv) {
    const counter = env.Counter.getByName("my-counter");
    const count = await counter.increment();
    return Response.json({ count });
  },
};

export class Counter extends DurableObject {
  private counter = 0;
  async increment() {
    return ++this.counter;
  }
}
```

## Environment variables

In addition to resource bindings, you can pass plain environment
variables via the `env` prop. These are separate from `bindings` and
appear directly on the `env` object at runtime:

```typescript
// alchemy.run.ts
export const Worker = Cloudflare.Worker("Worker", {
  main: "./src/worker.ts",
  env: {
    API_BASE_URL: "https://api.example.com",
  },
  bindings: {
    DB,
  },
});
```

Access them as `env.API_BASE_URL` in your handler. Note that `env`
values set this way are plain text or secret text bindings, not typed
through `InferEnv`.
